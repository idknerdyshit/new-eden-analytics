# New Eden Analytics

EVE Online market analytics platform ‚Äî correlates ship/item destruction with raw material price movements using cross-correlation and Granger causality.

## Architecture

Four services orchestrated by Docker Compose:

| Service | Port | Description |
|---------|------|-------------|
| `timescaledb` | 5432 | TimescaleDB (PostgreSQL 16) ‚Äî all persistent state |
| `backend-server` | 3001 | Axum HTTP API |
| `backend-worker` | ‚Äî | Background data ingestion + analysis |
| `frontend` | 3000 | SvelteKit SSR app |

## Directory Layout

```
backend/          Rust workspace (6 crates + sde-import + kill-backfill utilities)
frontend/         SvelteKit app
infra/docker/     Dockerfiles (Dockerfile.backend, Dockerfile.frontend)
```

## Quick Start

```bash
cp .env.example .env   # fill in ESI_CLIENT_ID, ESI_CLIENT_SECRET, SESSION_SECRET
make up                # docker compose up -d
make seed-sde          # one-time SDE import (~2-5 min)
# visit http://localhost:3000
```

## Makefile Commands

- `make up` / `make down` ‚Äî start/stop all services
- `make build` ‚Äî rebuild Docker images
- `make logs` ‚Äî tail all service logs
- `make seed-sde` ‚Äî import EVE static data (item types, blueprints)
- `make backfill-kills` ‚Äî backfill ~90 days of historical killmails from zKillboard + ESI (resumable)
- `make migrate` ‚Äî run DB migrations
- `make test` ‚Äî run backend unit tests (`cargo test --lib --bins`)
- `make clean` ‚Äî full reset: removes containers, volumes (destroys DB data), locally-built images, and orphans

## Environment Variables

Defined in `.env` (copy from `.env.example`):

- `DATABASE_URL` ‚Äî Postgres connection string
- `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB` ‚Äî used by TimescaleDB container
- `ESI_CLIENT_ID`, `ESI_CLIENT_SECRET`, `ESI_CALLBACK_URL` ‚Äî EVE SSO OAuth2
- `SESSION_SECRET` ‚Äî cookie signing key
- `RUST_LOG` ‚Äî tracing filter (default: `info`)

## Data Flow

1. **SDE Import** (one-time) ‚Äî item types + blueprints from Fuzzwork CSVs
2. **Market History** (hourly) ‚Äî daily OHLCV for The Forge from ESI
3. **Order Snapshots** (hourly) ‚Äî best bid/ask at Jita 4-4 from ESI
4. **Killmails** (continuous) ‚Äî destruction data from zKillboard R2Z2
5. **Daily Aggregation** (hourly) ‚Äî rolls up killmails into daily destruction volumes
6. **Correlation Analysis** (daily 02:00 UTC) ‚Äî cross-correlation + Granger causality over 180-day windows

## External APIs

- **ESI** (esi.evetech.net) ‚Äî no API key, rate-limited via `X-ESI-Error-Limit-Remain` header
- **R2Z2/zKillboard** ‚Äî no API key, sequential polling with User-Agent header
- **Fuzzwork** ‚Äî CSV downloads for SDE data, used only during seed

## Cross-Cutting Concerns

- Frontend proxies `/api/*` to backend: Vite dev proxy in development, `hooks.server.ts` in production
- `X-Request-Id` (UUID v4) generated by backend middleware, propagated through proxy to frontend API client
- All logging uses `[nea]` prefix on frontend, `tracing` crate on backend
